title: "Criar PixQrCode"

### Authorization

<ParamField path="Authorization" type="string" required>
  Cabeçalho de autenticação Bearer no formato `Bearer <api-key>` onde `<api-key>` é a sua chave de API.
</ParamField>

<RequestExample>

```bash Curl
curl --request POST \
  --url https://pay.valora.cash/v1/pixQrCode/create \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --body '{
  "amount": 123,
  "customer": {
    "name": "John Doe",
    "phone": "(11) 000-0000",
    "email": "johndoe@valorapay.cash",
    "tax_id": "123.456.789-01"
  },
}'
```

```typescript TypeScript
import axios from "axios";

                name: "Maria Oliveira",
                tax_id: "98765432100"
            }
        },
        {
            headers: {
                Authorization: `Bearer ${ApiKey}`,
                "Content-Type": "application/json"
            }
        }
    );

    console.log(Response.data);
}

CreatePix();
```

### Body

<ParamField path="amount" type="number" required>
  Valor da cobrança.
</ParamField>

<ParamField path=" customer" type="object" required>
  Os dados do seu cliente para criá-lo. O objeto de customer é obrigatório, mas ao informar qualquer informação do `customer` todos os campos `name`, `phone`, `email` e `tax_id` são obrigatórios.

  <Accordion title="Child attributes">
    <ParamField path="customer.name" type="string" required>
      Nome completo do seu cliente

      Example: `"John Doe"`
    </ParamField>
    <ParamField path="customer.phone" type="string" required>
      Celular do cliente

      Example: `"(00) 0000-0000"`
    </ParamField>
    <ParamField path="customer.email" type="string" required>
      Email do cliente

      Example: `"johndoe@valoracash.pay"`
    </ParamField>
    <ParamField path="customer.tax_id" type="string" required>
      CPF ou CNPJ do cliente

      Example: `"000.000.000-00"`
    </ParamField>
  </Accordion>
</ParamField>

### Response

<ParamField path="response" type="object">
  QRCode Pix criado com sucesso

  <Accordion title="Child attributes">
    <ParamField path="txid" type="string">
      Identificador único do QRCode Pix.
    </ParamField>
    <ParamField path="amount" type="string">
      Valor a ser pago
    </ParamField>
    <ParamField path="status" type="string">
      Informação sobre o andamento do QRCode Pix.

      Available options:  `Aguardando`, `Expirado`, `Cancelado`, `Paga`, `Reembolsado`
    </ParamField>
    <ParamField path="method" type="string">
      Informação sobre o método do QRCode Pix.

      Available options:  `PIX`,
    </ParamField>
    <ParamField path="pix_copia_e_cola" type="string">
      Código copia-e-cola do QRCode Pix.
    </ParamField>
    <ParamField path="created_at" type="string">
      Data de criação do QRCode Pix.
    </ParamField>
    <ParamField path="updated_at" type="string">
      Data de atualização do QRCode Pix.
    </ParamField>
    <ParamField path="expired_at" type="string">
      Data de expiração do QRCode Pix.
    </ParamField>
  </Accordion>
</ParamField>
