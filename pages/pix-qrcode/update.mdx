### Authorization

<ParamField path="Authorization" type="string" required>
  Cabeçalho de autenticação Bearer no formato `Bearer <api-key>` onde `<api-key>` é a sua chave de API.
</ParamField>

<RequestExample>

```bash Criar QRCode PIX
curl --request POST \
  --url https://pay.valora.cash/v1/pixQrCode/update \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --body '{
  "txid": "83106dc5928047f395b2b5d551c9b3c3"
  "amount": "10.0",
}'
```

### Query Parameters

<ParamField path="txid" type="string" required>
  ID do QRCode Pix
</ParamField>

<ParamField path="amount" type="number">
  Valor da cobrança que deseja mudar.
</ParamField>

<ParamField path="customer" type="object">
  Os dados do seu cliente para atualizá-los O objeto de customer não é obrigatório, mas todos os campos name, phone, email e tax_id são obrigatórios

  <Accordion title="Child attributes">
    <ParamField path="customer.name" type="string" required>
      Nome completo do seu cliente

      Example: `"John Doe"`
    </ParamField>
    <ParamField path="customer.phone" type="string" required>
      Celular do cliente

      Example: `"(00) 0000-0000"`
    </ParamField>
    <ParamField path="customer.email" type="string" required>
      Email do cliente

      Example: `"johndoe@valoracash.pay"`
    </ParamField>
    <ParamField path="customer.tax_id" type="string" required>
      CPF ou CNPJ do cliente

      Example: `"000.000.000-00"`
    </ParamField>
  </Accordion>
</ParamField>
